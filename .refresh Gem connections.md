For standard Gemini Gems (the custom AI versions created in the Gemini web app), the GitHub connection works as a snapshot.

Gemini Gems do not have real-time access to your GitHub repository. They only have access to the files and versions as they existed at the exact moment the repository was added to the Gem's knowledge base.

Here is a breakdown of how this works and how to manage it:

## 1. The Snapshot Mechanism

When you add a GitHub repository to a Gem's knowledge (or to a specific chat), Gemini essentially "downloads" a copy of the text from those files at that specific point in time.

No Auto-Sync: If you push new commits, merge branches, or add new files to GitHub after creating the Gem, the Gem will not know about them. It will continue to answer questions based on the old code.

Static Reference: The Gem does not maintain a live link to the repo; it maintains a reference to the uploaded context.

## 2. How to Update the Gem

Since there is no "Refresh" button for GitHub sources in a Gem, you must manually trigger an update if your code has changed significantly:

Edit the Gem: Go to your Gem manager and click the "Edit" (pencil) icon for that Gem.

Remove the Repo: Delete the existing GitHub repository entry from the Gem's knowledge base.

Re-add the Repo: Add the repository again. This forces Gemini to fetch the latest version of the files from GitHub.

Save: Save the Gem.
